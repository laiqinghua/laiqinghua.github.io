## 01

去年 Prompt 最火的时候，我做了一个提示词共享平台 PrompterHub.cn。半年多过去，平台积累了一万用户。

在 Skill 刚火起来的时候，我不只一次想在这个平台加入 Skill 模块，但 Skill 是托管文件夹，Prompt 是处理文本，前者对技术和资源要求高，靠我一人很难做起来。

不过 openclaw 的创始人 Peter Steinberger 就做了这个平台 ClawHub，一个面向 AI Agent 的 Skills 共享平台，目前平台上已有 10,322 个 Skill，涵盖了从企业 SaaS 到个人生产力的各个领域。

因为 openclaw 开源爆火的缘故，这个平台也成了最权威的 Skill 网站之一。

比如以前你想要把 YouTube 视频转成中文总结，得先找到合适的 Prompt，复制粘贴给 AI，再发视频链接，每次都要重复这套流程。但现在有了 Skill，你只需要装一次「YouTube 视频转中文」的 Skill，之后直接丢 Youtube 链接就行，AI 自动识别、自动处理、自动输出。

也可以到 ClawHub 网站探索一下，看看别人是怎么做 Skill 的。

我花了几天时间，分析了下载量前 1000 的 Skill，想弄明白一个问题：Skill 究竟可以应用到哪些领域？哪些场景已经被验证？未来 Skill 会不会终结传统的 SaaS 模式？

## 02

如果把 AI Agent 比作智能手机，那 ClawHub 就是它的 App Store。

简单来说，ClawHub 提供三件事：

**发布与版本化**。开发者可以将自己编写的 Skill 上传到 ClawHub，像发布 App 一样进行版本管理。用户安装时能明确知道这是 1.0 还是 2.0，更新日志写了什么。

**检索与发现**。目前平台已有 10,322 个 Skill，涵盖办公、开发、内容创作、金融等各个领域。用户可以通过分类、搜索、排行榜找到需要的 Skill。

ClawHub 是 Skill 的仓库，OpenClaw 是运行 Agent 的平台。

这种分工让 Skill 生态有了专业的基础设施。开发者专注写 Skill，用户专注用 Skill，平台负责连接双方。

## 03

我分析了 ClawHub 下载量前 1000 的 Skill，发现它们可以归纳为 6 大领域。每个领域的分布比例和核心逻辑，反映了当前 AI Agent 落地的真实需求。

对于每个领域，我各推荐 5 个热门 Skill，可以复制名称后面的链接，就能下载到 openclaw 或者 Claudecode。

### 领域一：企业集成与自动化（35%）

Agent 需要「手和脚」来打破数据孤岛，参与到真实的业务流转中。这是目前绝对的主流刚需。

这类 Skill 主打「替人跑腿」，让 Agent 深度介入真实业务系统。Google Workspace、Slack、Notion 等办公协同巨头占据统治地位；Zoho、Salesforce、HubSpot 等业务系统形成密集矩阵；API Gateway、n8n 等自动化中间件则扮演连接一切的「胶水」角色。

如果你在开发 B 端系统，也可以学习 Notion、Slack 一样，尽早开放 API，为 AI Agent 时代做好准备。

**热门 Skill 推荐**：

- **Gog**：Google Workspace 的终极 CLI 控制器，打通 Gmail、Calendar、DriveDocs。
- **Github**：开发者刚需，用 gh CLI 自动操作 Issue、PR 和 CI 流程。
- **Notion**：官方 API 接入，自动创建和管理页面、数据库。
- **API Gateway**：托管式 OAuth 授权，一键连通 Slack、HubSpot 等 100+ 应用。
- **Slack**：让 Agent 在 Slack 频道和私信中直接回复、响应表情。

### 领域二：搜索与研究（20%）

突破大模型的知识库时效限制，解决输入端的「原材料」质量问题。

全网检索（Tavily、Brave Search）、深度内容提取（YouTube 字幕抓取）、文档解析（PDF/OCR、Markdown 转换）构成完整的信息处理 pipeline。YouTube 相关的爬取与解析极其火爆，说明长视频文字化、摘要化是创作者和研究者的极高频需求。

**热门 Skill 推荐**：

- **Tavily Web Search**：专为 AI 优化的搜索引擎，直接返回精简、高相关性的结果。
- **Summarize**：通吃网页、PDF、图片、音频、YouTube 的终极总结工具。
- **Nano Pdf**：用自然语言指令直接编辑和处理 PDF。
- **Brave Search**：轻量级网页搜索和内容提取器。
- **YouTube Watcher**：快速抓取 YouTube 字幕，高密度吸收长视频知识。

### 领域三：开发者设施（15%）

开发者在教 Agent 如何「更聪明、更省钱、更像人」地工作。这部分是极客精神的最佳体现。

比如降本增效（Token 优化器、Smart Model Switching、Free Ride）、自主性与记忆（Self-improving-agent、Memory Manager）、身份与角色（AI Persona OS）。这些 Skill 试图为 Agent 建立认知闭环，让它能自我纠错、持续进化。

**热门 Skill 推荐**：

- **self-improving-agent**：主动捕获执行错误和用户纠正指令，让 Agent 在失败中持续进化。
- **Ontology**：构建类型化知识图谱，让不同 Skill 共享上下文状态。
- **Free Ride**：自动管理 OpenRouter 免费模型，极致降本。
- **Mcporter**：直接列表、配置和调用底层 MCP 服务器。
- **Model Usage**：精准统计 Token 开销和模型使用情况。

### 领域四：代码生成与环境（15%）

代码生成的最后一公里，需要具体的环境交互能力。

Next.js Expert、React Expert 等技能强制执行最佳实践；Docker、Git、tmux 等基础设施控制让 Agent 能直接操作底层环境；Frontend Design、Shadcn UI 等则面向「生成即用」的现代前端工作流。

**热门 Skill 推荐**：

- **Frontend Design**：极具审美的前端生成器，直接生成生产级精致界面代码。
- **SuperDesign**：提供专家级 UI 规范和约束。
- **Tmux**：让 Agent 远程控制 tmux 会话处理交互式命令行。
- **Opencode-controller**：用斜杠命令控制 Opencode 会话、切换代理和协同编程。
- **Debug Pro**：提供严谨的 7 步调试协议。

### 领域五：多媒体与内容创作（10%）

多模态能力的落地延伸，让 Agent 具备调用多模态模型的能力。

视觉（Nano Banana、Krea API）、听觉（ElevenLabs、Whisper）、文字（SEO Content Writer、Humanizer）全覆盖。

还有离钱和注意力最近的私人订制服务。

**热门 Skill 推荐**：

- **Nano Banana Pro**：调用 Gemini 3 Pro 进行高质量图像生成和定点编辑。
- **Humanizer**：抹除 AI 写作套路感，让文本更像人话。
- **Openai Whisper**：本地运行 Whisper 进行高精度语音转文字。
- **Humanize AI text**：重构大模型生成文本以绕过机器判别系统。
- **Video Frames**：自动调用 ffmpeg 从视频中精准提取关键帧。

### 领域六：垂直场景（5%）

Polymarket 预测市场、美股/A 股分析、量化回测等交易与研报工具，以及 ADHD Planner、习惯追踪、Garmin 运动分析等个人效率工具并存。这个领域虽然占比最小，但用户付费意愿最强。

**热门 Skill 推荐**：

- **Weather**：朴实无华但刚需的实时天气查询，无需 API Key。
- **Sonoscli**：用命令行直接指挥家里的智能音箱。
- **Polymarket**：接入全球预测市场数据源，查看事件赔率和走向。
- **Automation Workflows**（7.1k）：教 Agent 串联 n8n、Zapier 来解放双手。
- **Stock Market Pro**：基于 Yahoo Finance 的美股深度分析工具。

这 30 个头部 Skill 和 6 大领域分布，拼凑出了 Skill 的最佳应用领域。

用 API 打通信息流（35%），用研究解析工具获取高质量输入（20%），用开发者设施落地代码（15%），用运维优化工具持续进化（15%），用多媒体工具生产内容（10%），最后用垂直场景 Skill 服务具体需求（5%）。

这些 Skill 就反馈了 AI 现阶段能够完美落地的场景，对我的启发不只是可以把自己相关领域的工作 Skill 化，还可以产品化给大众用户，这或许在不久将来在每个领域都会有新的巨头的诞生。

## 04

ClawHub 的 Skill 使用非常简单，目前支持两种方式。

### 1 命令行安装（OpenClaw 用户）

如果你在使用 OpenClaw，可以直接通过命令行安装：

```
npx clawhub@latest install sonoscli
```

这种方式默认安装到 openclaw/skills 目录下，支持版本管理和回滚。安装后，Skill 立即生效，无需额外配置。

### 2 对话安装（Claude Code 用户）

如果你在用 Claude Code，安装更简单：

- 直接发给 Claude：「安装这个 Skill」
- Claude 会自动下载到 .claude/skills 目录并启用

**技巧：软链接同步**

如果你同时用 OpenClaw 和 Claude Code，可以用软链接让两个平台共享同一套 Skill：

```
ln -s ~/.claude/skills ~/openclaw/skills
```

这样在一个平台安装的 Skill，另一个平台也能直接用。

### 3 从 GitHub 导入

clawhub 还支持直接从 GitHub 仓库导入 Skill。只要仓库里有标准的 skill.json 配置文件，ClawHub 就能自动识别并安装。这比上传文件方便很多。

## 05

做完这个分析，我有一个越来越强烈的判断：Skill 正在吃掉 SaaS，或者说是订阅制的软件，尤其是工具类。

SaaS 不会消失，但它的使用方式和商业模式会被改变。

### 从「打开应用」到「直接说话」

传统 SaaS：你想记录想法，先打开笔记 App，打字输入完成。

Skill 模式：直接跟 AI 输入笔记，Agent 调用笔记记录 Skill，保存内容并同步到你的各个笔记软件里，甚至能根据你的文风，从笔记扩写一篇文章，制作一个视频等等。

### 从「订阅制」到「按需调用」

传统 SaaS：每月固定付费，不管你用不用。Notion、Slack、各种工具堆在一起，月费几百块。

Skill 模式：按调用付费，甚至免费。

### 从「功能堆砌」到「原子化能力」

传统 SaaS：每个产品都想做大而全，功能越堆越多，越来越臃肿。

Skill 模式：每个 Skill 只做一件事，但做到极致。Gog 只专注 Google Workspace 控制，Summarize 只专注内容提炼。

### 从「数据孤岛」到「上下文共享」

传统 SaaS：你在 Notion 写的文档，在 Slack 讨论的内容，在 Gmail 发的邮件，彼此割裂。

Skill 模式：Ontology Skill 构建知识图谱，让不同 Skill 共享上下文状态，形成统一的工作记忆。

未来的 SaaS 可能更像「基础设施提供商」，提供 API 和底层能力，而 Skill 开发者负责把这些能力封装成用户友好的交互方式。

就像 AWS 不会消失，但用户不会直接操作 AWS 控制台，而是通过 Skill 间接调用它的计算资源。
