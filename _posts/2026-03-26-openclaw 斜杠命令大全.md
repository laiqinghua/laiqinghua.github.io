## 01 斜杠命令是什么

斜杠命令是 OpenClaw Gateway 处理的聊天命令。大多数命令必须作为**独立消息**发送，以 `/` 开头。

主机专属的 bash 聊天命令使用 `! <cmd>`（`/bash <cmd>` 是其别名）。

有两个相关系统：

**命令**：独立的 `/...` 消息

**指令**：`/think`、`/fast`、`/verbose`、`/reasoning`、`/elevated`、`/exec`、`/model`、`/queue`
- 指令在模型看到之前会从消息中剥离
- 在普通聊天消息中（非纯指令消息），它们被视为"内联提示"，不会持久化会话设置
- 在纯指令消息中（消息仅包含指令），它们会持久化到会话并回复确认
- 指令仅对**授权发送者**生效

还有几个**内联快捷方式**（仅限授权发送者）：`/help`、`/commands`、`/status`、`/whoami`（`/id`）
它们会立即执行，在模型看到剩余文本之前被剥离，剩余文本继续通过正常流程处理。

## 02 完整命令列表

### 文本 + 原生命令（启用时）

`/help` - 显示帮助信息

`/commands` - 显示可用命令列表

`/skill <name> [input]` - 按名称运行技能

`/status` - 显示当前状态（包括当前模型提供商的使用量/配额）

`/allowlist` - 列出/添加/删除允许列表条目

`/approve <id> allow-once|allow-always|deny` - 解决 exec 批准提示

`/context [list|detail|json]` - 解释"上下文"；`detail` 显示每个文件/工具/技能/系统提示的大小

`/export-session [path]` - 别名 `/export`，将会话导出为包含完整系统提示的 HTML

`/whoami` - 显示你的发送者 ID；别名 `/id`

`/session idle <duration|off>` - 管理非活动自动取消聚焦

`/session max-age <duration|off>` - 管理硬最大年龄自动取消聚焦

`/subagents list|kill|log|info|send|steer|spawn` - 检查、控制或为当前会话生成子代理

`/acp spawn|cancel|steer|close|status|set-mode|set|cwd|permissions|timeout|model|reset-options|doctor|install|sessions` - 检查和控 ACP 运行时会话

`/agents` - 列出此会话的线程绑定代理

`/focus <target>` - Discord：将此线程或新线程绑定到会话/子代理目标

`/unfocus` - Discord：移除当前线程绑定

`/kill <id|#|all>` - 立即中止一个或所有运行中的子代理

`/steer <id|#> <message>` - 立即引导运行中的子代理

`/tell <id|#> <message>` - `/steer` 的别名

`/config show|get|set|unset` - 持久化配置到磁盘（仅限所有者）

`/debug show|set|unset|reset` - 运行时覆盖（仅限所有者）

`/usage off|tokens|full|cost` - 每条响应的使用量页脚或本地成本摘要

`/tts off|always|inbound|tagged|status|provider|limit|summary|audio` - 控制 TTS

`/stop` - 停止当前运行

`/restart` - 重启会话

`/dock-telegram` - 别名 `/dock_telegram`，切换回复到 Telegram

`/dock-discord` - 别名 `/dock_discord`，切换回复到 Discord

`/dock-slack` - 别名 `/dock_slack`，切换回复到 Slack

`/activation mention|always` - 仅限群组

`/send on|off|inherit` - 仅限所有者

`/reset` 或 `/new [model]` - 可选模型提示，其余文本作为消息体传递

`/think <off|minimal|low|medium|high|xhigh>` - 动态选择；别名 `/thinking`、`/t`

`/fast status|on|off` - 省略参数显示当前有效快速模式状态

`/verbose on|full|off` - 别名 `/v`

`/reasoning on|off|stream` - 别名 `/reason`

`/elevated on|off|ask|full` - 别名 `/elev`

`/exec host=<sandbox|gateway|node> security=<deny|allowlist|full> ask=<off|on-miss|always> node=<id>` - 发送 `/exec` 显示当前设置

`/model <name>` - 别名 `/models`

`/queue <mode>` - 加上选项如 `debounce:2s cap:25 drop:summarize`

`/bash <command>` - 仅限主机；`! <command>` 的别名

### 仅文本命令

`/compact [instructions]` - 压缩会话

`! <command>` - 仅限主机；一次一个

`!poll` - 检查输出/状态

`!stop` - 停止运行中的 bash 作业

## 03 使用技巧

### 命令格式

命令在命令和参数之间接受可选的 `:`，例如 `/think: high`、`/send: on`、`/help:`

### 模型选择

`/new <model>` 接受模型别名、`provider/model` 或提供商名称（模糊匹配）；如果不匹配，文本将作为消息体处理

示例：
```
/model
/model list
/model 3
/model openai/gpt-5.2
/model opus@anthropic:default
/model status
```

### 内联快捷方式

某些命令在嵌入普通消息中时也能工作，并在模型看到剩余文本之前被剥离

示例：`hey /status` 会触发状态回复，剩余文本继续通过正常流程处理

目前支持：`/help`、`/commands`、`/status`、`/whoami`（`/id`）

### 技能命令

`user-invocable` 技能被暴露为斜杠命令。名称被清理为 `a-z0-9_`（最多 32 个字符）；冲突会获得数字后缀（例如 `_2`）

`/skill <name> [input]` 按名称运行技能（当原生命令限制阻止每个技能的命令时很有用）

默认情况下，技能命令作为普通请求转发给模型

技能可以选择声明 `command-dispatch: tool` 以将命令直接路由到工具（确定性，无需模型）

## 04 配置选项

```json5
{
  commands: {
    native: "auto",
    nativeSkills: "auto",
    text: true,
    bash: false,
    bashForegroundMs: 2000,
    config: false,
    debug: false,
    restart: false,
    allowFrom: {
      "*": ["user1"],
      discord: ["user:123"],
    },
    useAccessGroups: true,
  },
}
```

**commands.text**（默认 `true`）启用聊天消息中的 `/...` 解析

**commands.native**（默认 `"auto"`）注册原生命令
- 自动：Discord/Telegram 开启；Slack 关闭（直到添加斜杠命令）；不支持原生命令的提供商忽略

**commands.nativeSkills**（默认 `"auto"`）原生注册**技能**命令
- 自动：Discord/Telegram 开启；Slack 关闭（Slack 需要为每个技能创建斜杠命令）

**commands.bash**（默认 `false`）启用 `! <cmd>` 运行主机 shell 命令

**commands.bashForegroundMs**（默认 `2000`）控制 bash 切换到后台模式前的等待时间

**commands.config**（默认 `false`）启用 `/config`（读取/写入 `openclaw.json`）

**commands.debug**（默认 `false`）启用 `/debug`（仅运行时覆盖）

**commands.allowFrom**（可选）设置每个提供商的命令授权允许列表

**commands.useAccessGroups**（默认 `true`）在未设置 `commands.allowFrom` 时强制执行命令的允许列表/策略

## 05 注意事项

### 授权

- 纯命令消息来自授权发送者时会立即处理（绕过队列 + 模型）
- 群组提及门控：来自授权发送者的纯命令消息绕过提及要求
- 未经授权纯命令消息会被静默忽略，内联 `/...` 令牌被视为普通文本

### 平台差异

**Discord**：
- 原生命令使用自动完成动态选项
- 省略必需参数时显示按钮菜单
- 原生命令 `/vc join|leave|status` 控制语音频道
- 线程绑定命令需要有效线程绑定启用

**Slack**：
- 仍支持 `channels.slack.slashCommand` 用于单个 `/openclaw` 风格命令
- 如果启用 `commands.native`，必须为每个内置命令创建一个 Slack 斜杠命令
- Slack 原生例外：注册 `/agentstatus`（而非 `/status`），因为 Slack 保留了 `/status`
- 文本 `/status` 在 Slack 消息中仍然有效

**Telegram**：
- 命令参数菜单在省略参数时显示按钮菜单

### 安全提醒

`/reasoning` 和 `/verbose` 在群组设置中有风险：它们可能暴露你无意透露的内部推理或工具输出

在群组聊天中最好保持关闭，尤其是 `/verbose`

## 06 实用示例

### 查看状态
```
/status
```

### 切换模型
```
/model list
/model 3
```

### 控制思考模式
```
/think high
/fast on
/verbose off
```

### 管理子代理
```
/subagents list
/kill all
/steer 1 请重新分析这个问题
```

### 调试配置
```
/debug show
/debug set messages.responsePrefix="[openclaw]"
/debug reset
```

### 持久化配置
```
/config show
/config set channels.whatsapp.allowFrom=["+1555","+4477"]
```
