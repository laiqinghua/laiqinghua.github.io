2025 年 11 月,一个 iOS 开发者在周末没事干,写了个小玩意：把 AI 大模型接到 WhatsApp 上,让它帮自己回消息。

他没想到,这个"周末项目"开源后 3 周,GitHub Stars 涨到了 123,000。

然后 Anthropic 来了封律师信,说项目名字"Clawdbot"跟 Claude 太像了,要求改名。项目先改叫 Moltbot,社区觉得不好听,继续投票,最终定名 OpenClaw。截至 2026 年 2 月,Stars 突破 227,000,Fork 超过 43,000,成为史上增长最快的开源项目之一。

这玩意到底解决了什么问题,让那么多人疯狂？

## 01 OpenClaw 是什么

一句话：跑在你自己电脑上的 AI 私人助手,通过 WhatsApp、Telegram、Discord 等你已经在用的聊天软件跟你沟通。

注意几个关键词：

**你自己的电脑**

不是云端,不是别人的服务器。你的聊天记录、偏好设置、工作文件,全部存在本地。没有订阅费,没有第三方偷看你的数据。

**你已经在用的聊天软件**

不需要装新 App,不需要改变使用习惯。你平时在哪聊天,AI 就在哪等着你。

**7×24 小时在线**

装好之后它一直跑在后台,你随时发消息,随时得到回复。

## 02 它能做什么

光说"AI 助手"太抽象。OpenClaw 内置了 50+ 技能,覆盖这些场景：

日常效率

- 帮你管邮件（Google Workspace 直连）、整理日历、创建待办
- 定时提醒（设好 Cron 就跑,出门不用开电脑）
- 总结长文章、网页内容,摘要直接发回聊天

开发者场景

- 管理 GitHub Issue 和 PR,直接在 Telegram 上关闭一个 issue
- AI 编程助手,聊天框里描述需求,它直接帮你写代码
- 监控服务器,异常自动通知你

知识管理 & 多平台统一入口

- 连接 Obsidian、Notion,AI 帮你搜笔记、整理内容
- WhatsApp 开始的对话,Telegram 上继续,AI 记得上下文,在所有平台都认识你

完整技能分类,你安装后可以按需启用：

| 类别 | 代表技能 | 功能 |
|------|----------|------|
| 办公效率 | gog / notion / trello | 邮件、日历、文档、任务管理 |
| 笔记管理 | obsidian / apple-notes / bear | 笔记搜索和整理 |
| 开发工具 | coding-agent / github / tmux | 编程助手、PR 管理、终端 |
| 任务管理 | apple-reminders / things-mac | 提醒事项、Things 看板 |
| 多媒体 | spotify-player / peekaboo / camsnap | 音乐控制、截图、拍照 |
| 实用工具 | weather / summarize / nano-pdf | 天气、摘要、PDF 处理 |
| 系统管理 | healthcheck / model-usage | 健康检查、用量统计 |

## 03 跟其他方案有什么不同

市面上 AI 助手工具不少,OpenClaw 的定位很明确：它不做企业客服,它是你的私人助手。

跟 Botpress 比：

| 维度 | OpenClaw | Botpress |
|------|----------|----------|
| 定位 | 个人 AI 助手 | 企业级聊天机器人平台 |
| 数据存储 | 全部本地 | 存在 Botpress 云端 |
| 对话设计 | 自然语言驱动,零配置 | 可视化流程编辑器 |
| 价格 | 完全免费开源（MIT） | 免费层有限,企业版收费 |
| 目标用户 | 开发者、技术爱好者 | 企业客服团队 |

Botpress 适合做客服机器人,有可视化的对话流程编辑器,适合非技术团队。OpenClaw 不做客服,它是帮你本人管邮件、写代码、控制智能家居的那种助手。

跟 Rasa 比：

Rasa 是传统 NLU 路线的代表,需要标注训练数据、定义意图和实体,学习曲线很陡。OpenClaw 直接站在大语言模型的肩膀上,不需要训练数据,`npm install` 然后跑起来就能用,完全两个方向。

跟自己写 Bot 比：

很多开发者会想：我直接调 OpenAI API + 写个 Telegram Bot 不就行了？可以,但你很快会撞上这些问题：

自建方案需要自己搞定：

```
├── 消息平台适配（每个平台 API 都不一样）
├── 多轮对话和上下文窗口管理
├── 流式输出（AI 回复实时显示）
├── 工具调用（让 AI 执行实际操作）
├── 记忆系统（AI 记住你的偏好）
├── 安全控制（防止未授权访问）
├── 多模型切换（不同任务用不同模型）
├── 模型故障转移（主模型挂了自动切备用）
├── 媒体处理（图片、音频、视频）
├── 守护进程管理（7×24 小时运行）
└── 持续跟进各平台 API 变更
```

OpenClaw 把这些全部搞定了,你只需要 `npm install -g openclaw && openclaw onboard`,5 分钟就有一个功能完整的 AI 助手。

跟云端 AI 助手（ChatGPT、Claude.ai）比：

数据都在你手上,不担心账号封禁,不交月费,可以自由切换模型。

OpenClaw 不适合的场景：企业客服、多租户 SaaS、完全不想碰命令行的用户、需要 100% 可用性的场景（本地机器可能关机）。OpenClaw 从设计上就是单用户、本地优先的工具。

## 04 核心架构,看这张图就够了

OpenClaw 的架构分三层：

```
消息平台层（Channels）
WhatsApp │ Telegram │ Discord │ Slack │ Signal │ 飞书 │ LINE │ Teams │ ...
↓
Gateway（网关）
负责接收消息、路由调度、安全控制
默认跑在 127.0.0.1:18789
┌──────────┬──────────┬──────────┐
│ 路由引擎 │ 会话管理 │ 记忆系统 │
│ 技能系统 │ 工具系统 │ Cron 调度 │
└──────────┴──────────┴──────────┘
↓
AI 模型提供商层
OpenAI │ Anthropic │ Google │ Ollama │ 通义千问 │ DeepSeek │ ...
```

**Gateway** 是核心,一个长期运行的后台服务（守护进程）。macOS 上用 launchd 注册,Linux 上用 systemd,开机自启,常驻后台。它负责：

- 管理所有消息平台的长连接（自动重连、心跳保活）
- 把消息路由给对应的 AI 助手（Agent）
- 调用 AI 模型,处理工具调用循环
- 把回复实时推送回消息平台

**Agent** 是 AI 助手实例。默认只有一个 `main` Agent,你也可以创建多个,让不同平台对应不同性格的 AI。

**Channel** 是消息平台的连接通道。目前支持的平台：

| 平台 | 认证方式 |
|------|----------|
| WhatsApp | 扫码配对 |
| Telegram | Bot Token |
| Discord | Bot Token |
| Slack | OAuth App |
| Signal | 手机号注册 |
| iMessage（BlueBubbles） | macOS 原生 |
| 飞书 / LINE / Zalo | 各自 API |
| Google Chat / Teams | Workspace 集成 |
| WebChat（内置） | 无需认证,装好即用 |

## 05 一条消息背后发生了什么

你在 WhatsApp 给 AI 发"帮我查一下明天的天气",背后经历了这些：

1. WhatsApp 收到消息,转成统一格式传给 Gateway
2. Gateway 判断：这条消息属于哪个 Agent？发送者有没有授权？
3. 会话管理器加载历史上下文 + AI 的记忆文件 + 当前启用的技能
4. AI 组装完整 Prompt,调用模型（比如 Claude Opus 4.6）
5. 模型决定调用"天气工具",获取实况数据,生成回复
6. 回复实时推送到 WhatsApp,你看到逐字出现的打字效果
7. 会话历史写入磁盘,如果 AI 决定记住什么,写入记忆文件

整个过程,你只是发了一条消息。这个设计有三个关键点：

- **串行执行** — 同一会话的消息排队处理,防止竞态条件
- **流式输出** — 不用等 AI 想完再发,实时看到回复
- **工具调用循环** — AI 可以连续调用多个工具,直到任务完成

## 06 AI 的记忆是怎么工作的

这是 OpenClaw 跟一般聊天机器人最本质的区别之一：它有持久记忆,而且你完全看得见、改得了。

没有向量数据库,没有 RAG 管道,没有 Embedding。AI 的记忆就是写在磁盘上的 Markdown 文件：

```
~/.openclaw/workspace/
├── MEMORY.md # 长期记忆（精选的重要信息）
├── SOUL.md # AI 的性格设定
└── memory/
    ├── 2026-02-26.md # 今天的日志
    ├── 2026-02-25.md # 昨天的日志
    └── ...
```

每次会话启动,AI 都会加载 `MEMORY.md`（你的偏好、重要决策）和最近几天的日志。它不是每次都从零开始,它"记得"你。

这个设计的好处：

- **可读** — 直接打开 Markdown 文件,看 AI 记住了什么
- **可编辑** — 不想让 AI 记住的东西,删掉那行就行
- **可版本控制** — 扔进 Git,追踪记忆变化
- **零依赖** — 不需要额外的数据库服务

你可以打开 `SOUL.md` 改写 AI 的性格——让它更严肃、更幽默,或者指定它总是用某种格式回复你。

## 07 支持哪些 AI 模型

OpenClaw 支持 29+ 模型提供商,主流的全部涵盖：

| 类型 | 代表 | 特点 |
|------|------|------|
| 国际主流 | OpenAI GPT、Anthropic Claude、Google Gemini | 能力最强 |
| 本地运行 | Ollama（免费）、vLLM | 数据不出本地,隐私最佳 |
| 国产模型 | 通义千问、DeepSeek、智谱 GLM、Kimi | 中文能力好 |
| 代理路由 | OpenRouter、LiteLLM | 一个 Key 用所有模型 |
| 企业云 | AWS Bedrock、Cloudflare AI Gateway | 企业级可用性 |

官方推荐 Anthropic Claude Opus 4.6,理由是工具调用准确率高、长上下文能力强,对 Prompt 注入的防御也更好。预算有限的话,Ollama 本地跑开源模型（Llama、Qwen）也完全够用,而且数据完全不出你的机器。

模型故障转移：OpenClaw 支持配置主模型和备用模型,主模型挂了自动切换,保证助手始终在线。

## 08 开始安装

### 第一步：确认环境

OpenClaw 需要 Node.js 22 或以上。先检查你的版本：

```bash
node --version # 需要 v22.x.x 或更高
```

如果版本不对,推荐用 nvm 来安装。

macOS：

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
source ~/.zshrc
nvm install 22
nvm alias default 22
```

Linux（Ubuntu/Debian）：

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs
```

Windows（强烈建议通过 WSL2 安装）：

```bash
wsl --install # 重启后在 Ubuntu 终端里按 Linux 步骤操作
```

### 第二步：安装 OpenClaw CLI

```bash
npm install -g openclaw@latest
# 验证安装
openclaw --version
```

### 第三步：运行引导向导

```bash
openclaw onboard --install-daemon
```

`--install-daemon` 会把 Gateway 注册为系统服务,开机自启,不用每次手动启动。向导会依次让你选 AI 模型提供商：

```
? Select your AI provider:
❯ OpenAI (GPT-5.2, GPT-5.2-mini)
  Anthropic (Claude Opus 4.6, Sonnet 4.6)
  Google (Gemini 3.1)
  Ollama (本地模型,免费)
  OpenRouter
```

然后输入 API Key（还没有的话去对应平台申请,想免费用的话安装 Ollama 后选本地模型）。Gateway 端口直接回车用默认值 18789,向导完成后看到这个就成功了：

```
OpenClaw is ready!
Open http://127.0.0.1:18789/ to start chatting.
```

### 第四步：打开 Control UI 开始聊天

浏览器打开 `http://127.0.0.1:18789/`,发一句"你好！请做一下自我介绍。",AI 会实时流式回复,文字一个一个蹦出来,整个链路就通了。

除了浏览器,OpenClaw 还有原生客户端：

| 平台 | 类型 | 额外能力 |
|------|------|----------|
| macOS | 菜单栏应用 | Voice Wake 语音唤醒、Talk Mode 语音对话、Canvas 可视化工作区 |
| iOS | 移动应用 | Voice Wake、摄像头、Talk Mode |
| Android | 移动应用 | Talk Mode、屏幕录制、摄像头 |

## 09 常用命令速查

```bash
openclaw status          # 查看 Gateway 运行状态
openclaw doctor          # 自动诊断问题
openclaw --version       # 查看版本
openclaw health          # 完整健康检查
openclaw models list     # 查看可用模型
openclaw channels list   # 查看已接入的消息平台
openclaw update          # 更新到最新版本
```

OpenClaw 采用日期版本号格式（比如 `v2026.2.24`）,几乎每天发布新版本。遇到问题先 `openclaw update` 基本能解决一半。

## 10 几个常见安装问题

**装完找不到 openclaw 命令？**

npm 全局 bin 目录没加入 PATH。执行：

```bash
echo 'export PATH="$(npm config get prefix)/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

**网络超时,安装失败？**

国内用户换淘宝镜像：

```bash
npm config set registry https://registry.npmmirror.com
npm install -g openclaw@latest
```

**Node.js 版本太低 / 端口被占用？**

```bash
nvm install 22 && nvm use 22
# 或者改端口：
openclaw config set gateway.port 18790
```

**安装完想自检一下？**

```bash
openclaw doctor
```

自动检查 Node 版本、Gateway 状态、模型连通性、各 Channel 配置,输出诊断报告,哪里有问题一目了然。

## 11 目录结构,装完长这样

```
~/.openclaw/
├── openclaw.json # 主配置文件（JSON5 格式,支持注释）
├── workspace/
│   ├── MEMORY.md # AI 的长期记忆
│   ├── SOUL.md # AI 人格设定
│   ├── memory/ # 每日记忆日志
│   └── skills/ # 自定义技能目录
└── agents/ # Agent 配置和会话历史
```

你随时可以打开 `SOUL.md` 改写 AI 的性格,打开 `MEMORY.md` 看它记住了什么（或者删掉你不想让它记住的东西）。这是 OpenClaw 相比云端 AI 服务最大的优势之一：完全透明,完全可控。
