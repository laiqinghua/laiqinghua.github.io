## openclaw多智能体实战

### 1. 查看当前openclaw 智能体清单

```shell
openclaw agents list
```

```shell
PS C:\Users\lqh> openclaw agents list

🦞 OpenClaw  2026.3.2 (85377a2) — Like having a senior engineer on call, except I don't bill hourly or sigh audibly.

Agents:
- main (default)  main：当前存在的代理名称（Agent Name），
                  (default)：说明这是默认代理（当未指定具体代理时，会使用这个）
  Workspace: ~\.openclaw\workspace 代理的工作目录路径（~ 代表你的用户主目录，
  即 C:\Users\lqh）实际路径是：C:\Users\lqh\.openclaw\workspace
  Agent dir: ~\.openclaw\agents\main\agent 该代理的配置 / 数据存储目录
  实际路径：C:\Users\lqh\.openclaw\agents\main\agent
  Model: bailian/qwen3.5-plus 该代理使用的 AI 模型是 “百炼 / 通义千问 3.5-plus”
  Routing rules: 0 该代理没有配置任何路由规则（路由规则用于将不同渠道 / 账号的请求定向到特定代理）
  Routing: default (no explicit rules)路由模式为默认模式，没有配置自定义的路由规则
Routing rules map channel/account/peer to an agent. Use --bindings for full rules.
Channel status reflects local config/creds. For live health: openclaw channels status --probe.
Routing: default (no explicit rules)：

```

那么看到这里其实就应该知道每一个智能体都需要 【名称、工作目录、数据存储目录、模型、路由规则】
