装好 OpenClaw 之后,大多数人做的第一件事是打开浏览器,在 http://127.0.0.1:18789 里跟 AI 聊两句。

然后关上电脑出门,AI 就没了。

这篇要解决这个问题：把 AI 接到你的手机上,不管你在哪,发条消息就能用。顺带也讲怎么换一个更合适的模型。

## 01 Channel：消息平台的连接管道

OpenClaw 用"Channel"来表示每一个消息平台连接。可以把它理解成一根双向水管——外部平台的消息从这头流进来,AI 的回复从那头流出去。

- 一个 Telegram Bot = 一个 Channel
- 一个 WhatsApp 账号 = 一个 Channel
- 一个 Slack App = 一个 Channel
- 一个飞书机器人 = 一个 Channel

你可以同时连多个 Channel,AI 在所有平台上共享同一套记忆和设置。出门用 WhatsApp,上班用飞书,完全没问题。

Channel 管理常用命令：

```bash
openclaw channels list        # 查看所有已配置的平台
openclaw channels status      # 查看各平台连接状态
openclaw channels capabilities # 查看各平台支持的功能
openclaw channels add <平台>   # 添加新平台
openclaw channels login <平台> # 绑定账号（如 WhatsApp 扫码）
openclaw channels logout <平台> # 登出
openclaw channels logs <平台>  # 查看平台日志（排查问题用）
openclaw channels resolve <平台> <标识> # 解析用户/频道标识
openclaw channels remove <平台> # 移除
```

## 02 支持的平台一览

| 平台 | 接入难度 | 备注 |
|------|----------|------|
| Telegram | 最简单 | 只需要 Bot Token,5 分钟搞定 |
| WhatsApp | 简单 | 扫码配对,跟 WhatsApp Web 一样 |
| Web Chat | 最简单 | 内置,Gateway 启动即可用 |
| Discord | 中等 | 需要创建 Developer Application |
| Slack | 中等 | 需要配置权限和事件订阅 |
| 飞书 | 中等 | 企业自建应用,需要管理员权限 |
| Signal / LINE / Teams | 中等 | 各有对应接入方式 |

新手建议从 Telegram 开始,流程最顺,出问题最少。

## 03 Telegram 接入（推荐新手先试）

Telegram 接入就两步：创建一个 Bot,把 Token 填进配置文件。

### 第一步：创建 Telegram Bot

1. 打开 Telegram,搜索 `@BotFather`
2. 发送 `/newbot`
3. 输入 Bot 名称（比如"我的 AI 助手"）
4. 输入 Bot 用户名（必须以 `_bot` 结尾,比如 `my_ai_helper_bot`）
5. BotFather 会返回一串 Token：`123456789:ABCdefGHIjklMNOpqrsTUVwxyz`

### 第二步：配置 OpenClaw

编辑 `~/.openclaw/openclaw.json`,在 `channels` 里加入：

```json
{
  "channels": {
    "telegram": {
      "botToken": "123456789:ABCdefGHIjklMNOpqrsTUVwxyz",
      // 或者用环境变量：export TELEGRAM_BOT_TOKEN="..."
      "groups": {
        "requireMention": true, // 群组里需要 @Bot 才触发
      }
    }
  }
}
```

### 第三步：重启 Gateway 测试

```bash
openclaw gateway --port 18789 --verbose
openclaw channels status
# 看到 telegram: connected 就成功了
```

打开 Telegram,找到你的 Bot,发一句"你好",AI 就会回你了。

### Polling 还是 Webhook？

| 模式 | 原理 | 网络要求 | 适用场景 |
|------|------|----------|----------|
| Polling（默认） | 主动拉取消息 | 无需公网地址 | 本地/个人使用 |
| Webhook | 平台主动推送 | 需要公网 HTTPS | 生产环境部署 |

有服务器的话可以切换成 Webhook,实时性更好。本地想测试 Webhook,用 ngrok 临时暴露端口：

```bash
ngrok http 18789
# 获得类似 https://abc123.ngrok.io 的地址,填进 webhookUrl
```

Bot 在群组里不回复？

Telegram Bot 默认开启隐私模式,在群组里看不到普通消息。在 BotFather 里发 `/setprivacy` → 选你的 Bot → 选 **Disable**,关掉即可。

## 04 WhatsApp 接入

WhatsApp 用扫码配对方式,跟 WhatsApp Web 的登录原理一样——非官方协议,个人使用没问题,不要用于大规模商业场景。

### 第一步：扫码配对

```bash
openclaw channels login whatsapp
```

终端会显示二维码,用手机 WhatsApp 扫码即可。扫完显示 `Connected!` 就配对成功,凭证自动保存在本地。

### 第二步：配置访问控制

```json
{
  "channels": {
    "whatsapp": {
      // 白名单：只有这些号码可以跟 Bot 对话
      "allowFrom": ["+8613800138000", "+8613900139000"],
      // 想让所有人都能用：allowFrom: ["*"],
    }
  }
}
```

DM 安全策略

OpenClaw 内置配对审批机制（DM Pairing）：陌生人第一次发消息,Bot 会回一个配对码,你手动批准才能对话。在配置里把 `dmPolicy` 设为 `"pairing"` 开启,设为 `"open"` 则不需要审批。个人使用建议保持 `pairing` 防止滥用。

```bash
openclaw pairing list              # 查看待审批的人
openclaw pairing approve whatsapp ABC123 # 批准
```

### 连接断了怎么办？

```bash
# 直接重新登录
openclaw channels login whatsapp

# 反复断的话,删掉旧凭证重新来
rm -rf ~/.openclaw/credentials
openclaw channels login whatsapp
```

## 05 Discord 接入

### 第一步：创建 Discord Application

1. 访问 discord.com/developers/applications
2. 点击 "New Application",输入名称
3. 进入 "Bot" 页面 → "Add Bot"
4. 开启 **"Message Content Intent"**（重要！不开则收不到消息内容）
5. 复制 Bot Token

### 第二步：邀请 Bot 到服务器

"OAuth2 → URL Generator" 页面：Scopes 选 `bot` 和 `applications.commands`,Bot Permissions 选 Send Messages、Read Message History、Use Slash Commands、Attach Files。复制链接用浏览器打开,选服务器,完成邀请。

### 第三步：配置 OpenClaw

```json
{
  "channels": {
    "discord": {
      "token": "YOUR_DISCORD_BOT_TOKEN",
      // 或用环境变量：export DISCORD_BOT_TOKEN="..."
      "commands": {
        "native": true, // 启用 Slash Commands
        "text": true,   // 同时支持文本命令
      },
      // allowFrom: ["user-id-1"], // 可选：限制哪些用户
      // guilds: ["guild-id-1"],   // 可选：限制哪些服务器
    }
  }
}
```

Bot 在线但不回复？

先确认 "Message Content Intent" 已开启,再确认 Bot 在该频道有读写权限。

## 06 Slack 接入

Slack 比 Telegram 多一步——需要配置事件订阅,让 Slack 知道往哪里推消息。

### 第一步：创建 Slack App

1. 访问 api.slack.com/apps
2. "Create New App" → "From scratch"
3. 输入名称,选择工作区

### 第二步：配置 Bot 权限

在 "OAuth & Permissions → Bot Token Scopes" 里添加以下权限：

- `chat:write` - 发送消息
- `channels:history` - 读取公开频道
- `groups:history` - 读取私有频道
- `im:history` - 读取私信
- `app_mentions:read` - 接收 @提及
- `files:read` / `files:write` - 读写文件

点 "Install to Workspace",复制 **Bot User OAuth Token**（以 `xoxb-` 开头）。

### 第三步：开启 Socket Mode（推荐）

在 "Socket Mode" 页面开启 Socket Mode,生成 **App-Level Token**（以 `xapp-` 开头）。Socket Mode 不需要公网地址,OpenClaw 跑在本地就能收到 Slack 消息。

### 第四步：配置事件订阅

在 "Event Subscriptions" 页面开启 Events,在 "Subscribe to bot events" 里添加：

- `message.channels` - 公开频道消息
- `message.groups` - 私有频道消息
- `message.im` - 私信消息
- `app_mention` - 被 @ 提及

如果用的是 Webhook 模式（非 Socket Mode）,还需要在 "Request URL" 里填入服务地址：`https://your-domain.com/webhook/slack`

### 第五步：配置 OpenClaw

```json
{
  "channels": {
    "slack": {
      "botToken": "xoxb-你的 Token",
      "appToken": "xapp-你的 Token",
    }
  }
}
```

或者用环境变量：

```bash
export SLACK_BOT_TOKEN="xoxb-xxx"
export SLACK_APP_TOKEN="xapp-xxx"
```

Bot 不响应？

99% 是事件订阅没配好。检查四个事件类型是否都加了,再用 `/invite @your-bot` 把 Bot 拉进频道。

## 07 飞书接入

飞书用企业自建应用方式,需要管理员账号。

### 第一步：创建飞书应用

1. 访问飞书开放平台 open.feishu.cn
2. 创建企业自建应用,添加"机器人"能力
3. 在"权限管理"里开启：`im:message`（收发消息）、`im:message.group_at_msg`（群聊@消息）、`im:resource`（文件）

### 第二步：配置事件订阅

在"事件订阅"里设置请求地址（格式：`https://your-domain.com/webhook/feishu`）,添加事件 `im.message.receive_v1`（接收消息）。

### 第三步：配置 OpenClaw

飞书通过 OpenClaw 的 extension 机制支持,获取 App ID 和 App Secret 后填入配置。详细参数参考 OpenClaw 官方文档的飞书专项说明。

## 08 模型配置：换一个更好的 AI

接好平台之后,很多人会发现默认模型不够满意——要么太贵,要么太慢,要么回答质量一般。

最快的方式：跑一遍引导向导

向导会帮你选提供商、填 API Key、测试连通,三分钟搞定。

```bash
openclaw onboard
```

### 查看和切换模型

```bash
openclaw models status          # 当前用的什么模型
openclaw models list            # 列出所有可用模型
openclaw models set-image       # 单独设置图像模型
openclaw models aliases list    # 查看模型别名
openclaw models fallbacks list  # 查看故障转移列表
```

换模型只需一行：

```bash
openclaw models set "anthropic/claude-sonnet-4-6"
```

### 主流提供商和 API Key 获取

| 提供商 | 获取地址 | Key 格式 | 环境变量 |
|--------|----------|----------|----------|
| OpenAI | platform.openai.com | `sk-proj-...` | `OPENAI_API_KEY` |
| Anthropic | console.anthropic.com | `sk-ant-...` | `ANTHROPIC_API_KEY` |
| Google | aistudio.google.com | `AIzaSy...` | `GEMINI_API_KEY` |
| OpenRouter | openrouter.ai | `sk-or-...` | `OPENROUTER_API_KEY` |
| xAI (Grok) | x.ai | `xai-...` | `ZAI_API_KEY` |

有 Claude Max 订阅？可以不用 API Key 直接用 OAuth 认证,用你的订阅额度,不需要额外付 API 费用：

```bash
openclaw auth login anthropic
# 按提示完成授权,Token 自动存好
```

嫌多个 Key 麻烦？用 OpenRouter 一个 Key 访问所有主流模型,统一计费：

```bash
export OPENROUTER_API_KEY="sk-or-xxxxx"
openclaw models set "openrouter/anthropic/claude-sonnet-4-6"
```

### API Key 安全提示

不要把 Key 直接写进配置文件,用环境变量更安全：

```bash
# 加到 ~/.bashrc 或 ~/.zshrc
export OPENAI_API_KEY="sk-proj-xxxxx"
export ANTHROPIC_API_KEY="sk-ant-xxxxx"
export GEMINI_API_KEY="AIzaSy-xxxxx"
source ~/.bashrc
```

### 按场景选模型

| 我的需求 | 推荐模型 | 理由 |
|----------|----------|------|
| 日常闲聊、简单问答 | GPT-5.2-mini / Claude Haiku 4.5 | 便宜快,够用 |
| 编程、复杂分析 | Claude Sonnet 4.6 | 编程能力强,性价比高 |
| 中文场景 | Qwen-Max / Kimi-Max | 中文理解更到位 |
| 完全免费、隐私优先 | Ollama + llama3.1 / qwen2.5 | 数据不出本地 |
| 不想管多个 Key | OpenRouter | 一个 Key 统一调度 |

### 免费用本地模型：Ollama

没有预算、或者对数据隐私有要求,用 Ollama 在本地跑开源模型。8GB 内存可以跑 7B 参数模型,效果不输两年前的 GPT-3.5。

```bash
# 安装 Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 下载模型（选一个）
ollama pull llama3.1    # 综合能力强
ollama pull qwen2.5     # 中文最强开源模型
ollama pull phi3        # 轻量,低配也能跑

# 配置 OpenClaw 使用 Ollama
openclaw models set "ollama/qwen2.5"
```

### 配置故障自动转移

主模型挂了,自动切到备用模型。最后一个建议放本地模型,保证所有云服务都挂了还能用：

```json
{
  "agent": {
    "model": "anthropic/claude-sonnet-4-6",
    "modelFallback": [
      "openai/gpt-5.2",
      "google/gemini-2.5-flash",
      "ollama/llama3.1"  // 最后兜底用本地模型
    ],
  }
}
```

### 设置模型别名（可选）

给常用模型起个短名字,方便快速切换：

```bash
openclaw models aliases add coding "anthropic/claude-opus-4-6"
openclaw models aliases add cheap "openai/gpt-5.2-mini"

# 切换时直接用别名
openclaw models set coding
```

## 09 给 AI 设定人格

接好平台,配好模型,最后一步：让 AI 真正像你的"私人助手",而不是千篇一律的机器人。

编辑 `~/.openclaw/workspace/SOUL.md`：

```markdown
# 我的 AI 助手
你是我的私人助手,叫小助。

## 性格
- 简洁,不废话
- 有点幽默,但不过分
- 中文优先

## 擅长
- 帮我管待办事项
- 回答技术问题
- 总结长文

## 记住
- 不确定的事情直说不确定
- 重要信息主动记录下来
```

重启 Gateway 后生效。这是 AI 的"灵魂",它会按这里的设定来跟你互动。

## 11 常见问题

**Gateway 重启后平台断连？**

WhatsApp 偶尔需要重新扫码,Telegram 和 Discord 不需要。运行 `openclaw channels status` 查看,断了就 `openclaw channels login <平台>` 重连。

**消息发出去 AI 没回？**

用 `openclaw gateway --port 18789 --verbose` 前台启动,看实时日志定位问题。也可以用 `openclaw channels logs <平台>` 查看详细日志。

**Telegram 群组里 @Bot 没反应？**

检查两件事：BotFather 里 Privacy Mode 是否已关闭；`openclaw.json` 里 `groups.requireMention` 是否设置正确。

**Slack Bot 不响应？**

99% 是事件订阅没配好。检查 Event Subscriptions 里的四个事件类型是否都选了（`message.channels`、`message.im`、`app_mention`）,再用 `/invite @your-bot` 确认 Bot 在频道里。

**Webhook 收不到消息？本地怎么测试？**

用 ngrok 临时暴露本地端口：`ngrok http 18789`,把生成的 HTTPS 地址填进各平台的 Webhook URL。调试完再换成正式服务器地址。

**API Key 报错无效？**

Key 只在创建时显示一次,复制时注意不要多复制空格。用 `openclaw models status` 测试 Key 是否有效。

## 下一篇预告：技能系统和记忆系统

现在你的 AI 已经能在手机上用了,但它还是个"健忘症患者"——聊完就忘,每次都要重新介绍自己。

技能系统——给 AI 装上新能力

搜索网页、读写文件、调用 API、执行代码……默认的 AI 只会聊天,装上技能之后它能帮你真的干活。

记忆系统——让 AI 真正认识你

工作区记忆、用户记忆、对话记忆,三种记忆机制分别是什么、什么时候用、怎么让 AI 主动记住重要信息。

装好了这两样,你的 AI 助手才算真正"活"了。
